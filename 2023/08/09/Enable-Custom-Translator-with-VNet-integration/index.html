# Enable Custom Translator with VNet integration

# 1. 建立新的 Translator 資源

# 1.1 請先建立一個 Public 的 Translator 資源，確認 Allow public access

# 1.2 同時建立 Private endpoint

# 2. 在 Custom Translator Portal 先將 Workspace 加入

# 2.1 建立 Workspace

# 2.2 取得 Workspace ID

# 3. 確認 VNet 中建立一個 VM 可以連線到 Private endpoint

# 4. 將 Custom Translator 啟用為 VNet integration

# 4.1 將 Firewalls and virtual networks 設定為 Selected networks and private endpoints, 並加入 #3 VM 所在的 subnet, 加入完成後，Customer Translator 便啟用了 VNet integration (全部完成設定後可以改為 Disabled)

參考: Enable Custom Translator through Azure Virtual Network

# 5. 登入 VM 開啟 Command Prompt, 執行以下的命令，將 workspace 重新建立為 VNet integration

# 5.1 建立 workspace, 其中 workspace 名稱需要與 #2.1 建立的名稱相同

// create workspace
curl.exe  -X POST "https://<resource-name>.cognitiveservices.azure.com/translator/customtranslator/api/texttranslator/v1.0/workspaces" --header "Content-Type: application/json" -H "Ocp-Apim-Subscription-Key:<resource-key>" --data "{ 'Name': '<workspace-name>', 'Subscription': { 'SubscriptionKey': '<resource-key>', 'BillingRegionCode': '<billing-region-code>' } }"

# 5.2 取得支援的語言代碼

// get supported LanguagePairId code
curl.exe -X GET   "https://<resource-name>.cognitiveservices.azure.com/translator/customtranslator/api/texttranslator/v1.0/languages/supportedlanguagepairs" --header "Content-Type: application/json" -H "Ocp-Apim-Subscription-Key:<resource-key>"

# 5.3 建立 project, 其中 workspace id 為 #2.2 取得的 workspace id, sourceLanguageCode 與 targetLanguageCode 可以參考 #5.2 取得支援的語言代碼

// get supported LanguagePairId code
curl.exe -X GET   "https://<resource-name>.cognitiveservices.azure.com/translator/customtranslator/api/texttranslator/v1.0/languages/supportedlanguagepairs" --header "Content-Type: application/json" -H "Ocp-Apim-Subscription-Key:<resource-key>"

# 5.4 在 VM 開啟 Custom Translator Portal, 確認已經可以看到 #5.3 建立的 project
