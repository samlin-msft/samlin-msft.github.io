# Gathering CycleCloud service JVM heap usage to Log Analytics

# 準備事項

CycleCloud Service 使用 Java 開發，因此在安裝 CycleCloud Service 時，會自動安裝 Java 執行環境。而 Java 執行環境的記憶體使用量，可以透過 JMX (Java Management Extensions) 來取得。而 Heap 使用的資訊，可以透過 jstat 命令來取得。因此我們需要先安裝一些必要的套件，才能夠將 CycleCloud 的 JVM heap usage 資訊傳送到 Log Analytics 中。

  1. 安裝必要的套件

    (1) java-1.8.0-openjdk-devel 套件
    sudo yum install java-1.8.0-openjdk-devel

    (2) Python 使用 Azure SDK 相關的套件

    sudo pip3 install azure-identity
sudo pip3 install azure-keyvault-secrets
sudo pip3 install azure-monitor-query
sudo pip3 install azure-core

  2. 準備相關的服務與權限

    (1) CycleCloud 需要啟用 Managed Identity

    (2) Log Analytics workspace 並取得 Workspace ID 與 Primary Key 儲存於 Key Vault 中

    (3) Key Vault 存放 Log Analytics workspace ID 與 Primary Key

# 設定 CycleCloud Service JVM heap usage 收集到 Log Analytics

  1. 登入 CycleCloud Server, 確認 jstat 可以正確取得 heap usage 資訊

    jstat -gcutil <pid>

  1. 登入 CycleCloud, 建立 Custom Script 目錄
    mkdir ~/scripts

  2. 建立一個 蒐集 JVM heap usage 的 python 檔案

    (1) 建立一個 python 檔案

    vi ~/scripts/jsvc_heap_usage.py

    (2) 複製以下內容到 jsvc_heap_usage.py 檔案中，並修改以下的設定:

    • key_vault_url: Key Vault 的 URL
    • workspace_id_kv_secret: Key Vault 中存放 Workspace ID 的 Secret 名稱
    • workspace_key_kv_secret: Key Vault 中存放 Workspace Primary Key 的 Secret 名稱
    • table_name: Log Analytics 中的 Table 名稱

  1. 設定一個每一分鐘執行一次的 cron job

    (1) 設定 crontab

    crontab -e

    (2) 加入一行

    * * * * * /usr/bin/python3 ~/scripts/jsvc_heap_usage.py

    (3) 確認 crontab 設定成功

    crontab -l

    (4) 確認 crontab 執行結果

    sudo grep CRON /var/log/cron

  1. 確認 Log Analytics 資料

    (1) 開啟 Log Analytics 確認 Custom Table 已經產生

(2) 開啟 Custom Table 查看資料


(3) 查詢輸出為 Chart

# jstat 輸出說明

欄位名稱說明
S0C第一個 Survivor 區的容量（KB）。
S1C第二個 Survivor 區的容量（KB）。
S0U第一個 Survivor 區的使用量（KB）。
S1U第二個 Survivor 區的使用量（KB）。
ECEden 空間的容量（KB）。
EUEden 空間的使用量（KB）。
OC老年代（Old Gen）的容量（KB）。
OU老年代（Old Gen）的使用量（KB）。
MC元數據區（Metaspace）的容量（KB）。
MU元數據區（Metaspace）的使用量（KB）。
CCSC壓縮類空間（Compressed Class Space）的容量（KB）。
CCSU壓縮類空間（Compressed Class Space）的使用量（KB）。
YGC年輕代垃圾回收次數。
YGCT年輕代垃圾回收所花費的時間（秒）。
FGC老年代垃圾回收次數。
FGCT老年代垃圾回收所花費的時間（秒）。
GCT垃圾回收總共所花費的時間（秒）。
